Key user decisions (2026-07-09, don't relitigate):
- **PufferLib: fork the 4.0 C/CUDA branch** + `ocean/tyranny/` C socket-shim → Python `env_server.py` hosting CRPGEnv. User chose this knowing 4.0 is lockstep/Linux-only/no-throughput-gain; trainer side runs on WSL2 or Linux. NOT pufferlib 3.0 (user rejected), no GymnasiumPufferEnv deliverable.
- Faction reward = **in-game Favor deltas** for target faction, counted only during dialogue mode; tags only safeguard paraphrases.
- **Infrastructure-vs-gameplay split (REVISED AGAIN 2026-07-09, latest wins): character BUILD CHOICES are IN the agent's loop as STRUCTURED decisions; menu mechanics are not.** The agent picks build options (Conquest choices, background, class, attribute/skill spreads, combat style) via the SAME number-key actions used for dialogue — the adapter routes key i in CREATION mode to "select option i of the current stage"; the env applies the choice via the game's internal setters (not cursor pixel-hunting) and advances the stage (PressOkay). Name/appearance/portrait/voice/difficulty = env-scripted. Other infra stays scripted: start game, skip logos, confirm dialogs, pause, death recovery, loading detection, GUARD THE QUIT/ESC MENU (a random agent quit the game mid-episode via Escape menu — bridge died). Save-load reset (act1_save, RL1 save, ~5s) remains the fast debug path; the creation-in-the-loop path is the thesis-faithful episode start. Reflection-based creation COMPLETION still NREs (ConquestLocation etc.) — completion must go through the real flow once choices are applied; the recorded flow (tools/session_recording.jsonl) shows stage 0-6 + ready=True gate works when stages are genuinely completed. Level-ups: same structured-choice pattern later (task #14).
- Observation: high-res pixels (dialogue text legible in image; no text side-channel) + structured state + mode + goal.
- Dialogue randomizer: per-episode procedural rewording via the C# mod, invisible to agent, to prevent reward hacking; hybrid paraphrase bank (offline Claude Batches) + `--live-paraphrase` eval flag.
- v1 parallelism: **single instance only** (user parked multi-instance 2026-07-09 after second instance hit a Unity 'Fatal error' — log-file contention suspected, -logFile fix untested). Keep env-var port/save-dir hooks so it's resumable.
- C# bridge: BepInEx 5.4.22, net35, HarmonyX, TCP JSON IPC on 5555+instance_id.
- **Two mandatory BepInEx.cfg edits** (see bridge_mod/INSTALL.md, learned by debugging 2026-07-09): `HideManagerGameObject = true` (engine sweeps the plugin object) and `[Preloader.Entrypoint] Type = Camera` (default Application entrypoint force-loads game assemblies before MonoManager → "referenced script missing" on the Global prefab → game stuck on black SplashScreen, saves unloadable).
- Input injection strategy PROVEN: Harmony prefixes on 9 UnityEngine.Input icalls (mousePosition, buttons, keys, axes) — works on this old Mono, virtualizes input for both GameInput and NGUI UICamera in one seam.

**Direction (2026-07-10, approved plan `C:\Users\okohe\.claude\plans\i-need-you-to-encapsulated-cloud.md`):** finishing the RLE via 5 workstreams. Governing principle: **infrastructure ≠ gameplay** — start/load/difficulty/cutscene/dialog/pause/**death recovery+checkpoint**/loading/save/appearance are SCRIPTED; a predefined config (background/attributes/skills/combat-style/party/level-up/equipment/consumables/spells/formation/talents) is applied by the env via **scripted triggers fired when each is needed** (e.g. detect level-up screen → apply predefined choice). **Agent must NEVER reach the console to cheat** (hard lockdown). **PREFER the real in-game UI** to apply config; console only as locked-down fallback where no setter-shaped UI exists (equipment/formation/consumables/spells). Key verified facts: level-up REUSES the creation UI (`UIPartyPortraitIcon.OnLevelUpClick` → `UICharacterCreationManager.OpenCharacterCreation`, gated !InCombat) so `CreationChoices`/`EnumSetter`/`UICharacterCreationSkillSetter` scaffolding extends directly; `CharacterStats.LevelUpAvailable()` detects pending; `UIDeathManager.OnRespawnClicked` heals party + clears PartyDead/GameOver (death-recovery); quit sinks are `Game.GameState.LoadMainMenu(bool)` + `Application.Quit()` (patch both, gated on agent-active, to stop the historical agent-quits-via-Esc bridge kill). Metrics: `info` is discarded in `env_server.py`; plan bumps wire proto to v2 with a fixed float trailer → C `Log` fields → `binding.c my_log` → PufferLib native logging. `death_mode ∈ {terminal,revive,checkpoint}` default checkpoint (watchable early curriculum), keep `terminal` for benchmark comparability.
